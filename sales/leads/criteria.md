# ICHI 営業先選定基準（criteria.md）

> このファイルは Web リサーチ営業スクリプトが読み込む「ターゲット定義」です。
> Claude API はこの基準に従って企業を選別・スコアリング・下書き生成します。

## ターゲット概要
- 対象業態: ① 個人経営のカフェ・飲食店 ② 個人経営の美容室・サロン・ネイルサロン
- 規模: 個人経営（オーナー = 決裁者）。チェーン・FC本部は除外
- 提供サービス: 5万円〜の小さな業務自動化（ヒアリングで内容を決定・拡張）
- 当面のゴール: 受注より「最初の1事例」を作ること。PoC・お試し歓迎
- アプローチ手段: 公式サイトの問い合わせフォーム / 公開された問い合わせ窓口のみ

## なぜこのターゲットか
- 即決力: 個人経営はオーナーが決裁者。稟議がなくその場で決まる。実績ゼロが最初の1件を取る追い風。
- 事務作業の重さ: 専任の事務・IT担当がおらず、オーナーが片手間で予約管理・口コミ返信・シフト・帳簿等をこなす → 潜在ニーズが明確。
- 母数: 両業態とも個人事業者数が桁違いに多く、Web に情報を公開する店が多い。数を当てる戦略と相性が良い。

## 入口となる「あるある業務」（提案のフック）
### カフェ・飲食店
- Google マップ / 食べログ等の口コミ返信文作成
- 予約管理・予約確認やリマインドの連絡
- SNS 投稿文・メニュー説明文の作成
- 仕入れ・在庫のエクセル管理
- 問い合わせ（電話・メール）の一次対応
### 美容室・サロン・ネイルサロン
- 予約管理・予約リマインド・キャンセル対応
- 口コミ返信・リピート促進の DM 作成
- 顧客カルテ・来店履歴の管理
- SNS 投稿（施術写真の説明文など）の作成
- 新規メニュー・キャンペーン告知文の作成

## 相性スコアリング基準（1〜5、高いほど優先）
- 5: 個人経営が明確 + 公式サイトに問い合わせ窓口あり + Web集客に積極的（予約サイト/SNS活発）
- 4: 個人経営 + 問い合わせ窓口あり
- 3: 個人経営らしいが問い合わせ窓口が不明瞭（SNS DMのみ等）
- 2: 個人/チェーンの判別が曖昧
- 1: チェーン・FC の可能性が高い、または情報が乏しい
スコア 4 以上のみを下書き生成の対象とする。

## 除外条件
- チェーン店・フランチャイズ・大手資本の店舗
- 公式サイト・公開された問い合わせ窓口が一切ない店
- 個人名のみで企業実体が確認できないもの
- 出典 URL が確認できない（Claude が検索結果なしに名前を出した）もの → 必ず除外

## アプローチの訴求軸（下書き生成の方針）
1. 小さく試せる: 「まず1つの作業だけ、5万円から」
2. 安い: 相場より低い価格帯であることを明示
3. 現場が分かる: サービス業出身者として現場の忙しさを理解している姿勢
4. オーナーの時間を取り戻す: 「事務作業を手放して接客と現場に集中できる」
### 下書きのトーン
- 売り込み臭を抑え、相手の負担への共感から入る
- 専門用語（AI/API/自動化の技術用語）を避け、日常語で
- 「無料で一度お話を聞かせてください」など心理的ハードルの低い CTA
- 長すぎない（問い合わせフォーム想定なので 300〜400 字程度）

## 禁止事項（コンプライアンス）
- 個人のメールアドレスを収集・保存・送信先にしない
- スクレイピング（サイト自動巡回でのデータ抜き取り）は行わない
- 公開されている「問い合わせフォーム URL」のみを記録する
- 送信（フォーム投稿）は必ず人間が手動で行う。自動投稿はしない
- すべての企業情報に出典 URL を必須化する（捏造防止）

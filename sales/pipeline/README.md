# ICHI 営業パイプライン — Google Apps Script セットアップ

## 前提

- Google スプレッドシートに **"pipeline"** という名前のシートが存在すること
- ヘッダー行（1行目）に以下の列名が含まれていること（順序は問わない）

| 列名 | 内容 |
|------|------|
| 顧客名 | 商談相手の企業名 |
| 案件名 | 案件・プロジェクト名 |
| ステータス | 例: 初回接触 / 提案中 / 交渉中 / 受注 / 失注 |
| 次回アクション | 次に取るべき行動の内容 |
| 期日 | 次回アクションの期限（Date 型セル） |

## セットアップ手順

### 1. Apps Script エディタを開く

スプレッドシートのメニューから  
**拡張機能 → Apps Script** を選択

### 2. Code.gs を貼り付ける

エディタに表示されている既存のコードをすべて削除し、  
`Code.gs` の全文をペーストして **保存（Ctrl+S）**

### 3. setupTrigger を 1 回手動実行する（権限付与）

1. エディタ上部の関数セレクタで **`setupTrigger`** を選択
2. **▶ 実行** をクリック
3. 「権限の確認」ダイアログが出たら **許可**
4. 完了するとスプレッドシート側にアラートが表示される

> この手順でメール送信・スプレッドシート読み取り・トリガー作成の権限がまとめて付与されます。

### 4. 動作確認（任意）

スプレッドシートのカスタムメニュー **「ICHI 営業」→「今すぐリマインダー実行」** から手動実行できます。  
期日が明日以前かつ未クローズの行があればメールが届きます。

### 5. 以降の運用

翌日 9:00 JST から `dailyCheck` が自動実行されます。  
- 要対応案件がある日のみメールが届きます
- 要対応案件がゼロの日はメール送信をスキップし、ログのみ記録します

## トリガーの再設定

列追加などでシートを変更した場合、トリガーの再設定は不要です。  
列名でマッピングしているため、列の順序変更にも自動で対応します。

トリガーを一度削除して再設定したい場合は、カスタムメニューの **「トリガー設定」** を実行してください。

## メール仕様

- **送信先**: スクリプト実行ユーザーのアカウント
- **件名**: `[ICHI 営業] 本日の要対応 N件`
- **対象条件**:
  - ステータスが「受注」「失注」以外
  - 期日が今日 + 1日（明日）以前
- **ソート**: 期日超過が大きい順（緊急度の高い案件が上）
